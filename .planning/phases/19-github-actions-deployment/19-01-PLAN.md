  <action>
Add a `docs` job to `.github/workflows/publish.yml` after the existing `publish` job with these specifications:

**Job configuration:**
- Job name: `docs`
- Dependencies: `needs: [test, e2e]` (runs in parallel with `publish`)
- Runner: `runs-on: ubuntu-latest`
- Permissions: `permissions: contents: write` (required for gh-pages push)

**Steps (in order):**
1. Checkout repository: `actions/checkout@v4`
2. Setup Node.js: `actions/setup-node@v4` with `node-version: '20'` and `cache: 'npm'`
3. Install dependencies: `npm ci`
4. Build documentation: `npm run docs:ci`
5. Deploy to GitHub Pages: `peaceiris/actions-gh-pages@v4` with:
   - `github_token: ${{ secrets.GITHUB_TOKEN }}`
   - `publish_dir: ./docs`
   - `force_orphan: true`

**Critical constraints from CONTEXT.md:**
- Self-contained job (own checkout and npm ci, no artifact sharing)
- Non-blocking by default (job failure doesn't affect other jobs)
- No `continue-on-error` or retry logic
- No `if: always()` - docs should only deploy after tests pass
