  <name>Enable GitHub Pages in repository settings</name>
  <files></files>
  <action>
GitHub Pages must be enabled before the docs deployment workflow can succeed.

**Steps:**
1. Go to https://github.com/KrisGray/opencarl/settings/pages
2. Under "Build and deployment" → "Source", select "Deploy from a branch"
3. If gh-pages branch exists (from a previous workflow run): Select branch: gh-pages, folder: / (root)
4. If gh-pages branch doesn't exist yet: 
   - First, merge the publish.yml changes to trigger the workflow
   - Wait for the docs job to complete (creates gh-pages branch)
   - Then return to Pages settings and select gh-pages branch
5. Click Save

**Expected outcome:** Pages shows "Your site is live at https://krisgray.github.io/opencarl/"
