## Structure:
1. Quick Diagnosis Checklist (5 common checks)
2. Common Problems section with Problem → Diagnosis → Solution format
3. Debug Mode section (how to use CARL_DEBUG=true)
4. Getting Help section (when to file issues)

## Problems to Cover (with real examples):

### Installation Issues
- Problem: Plugin not loading after install
  - Diagnosis: Check opencode.json plugins array
  - Solution: Verify plugin path format, show correct config snippet
- Problem: Setup command not found
  - Diagnosis: Plugin not properly installed
  - Solution: Reinstall steps with commands

### Rule Loading Issues  
- Problem: Rules not appearing in context
  - Diagnosis: Check manifest STATE, recall keywords
  - Solution: Debug checklist with CARL_DEBUG=true example
- Problem: Project rules not loading
  - Diagnosis: Project rules require opt-in
  - Solution: How to enable project rules
- Problem: Domain file ignored
  - Diagnosis: Wrong filename format
  - Solution: Naming convention (lowercase, no extension)

### Matching Issues
- Problem: Wrong rules loading
  - Diagnosis: Overly broad recall keywords
  - Solution: Keyword specificity tips with before/after examples
- Problem: Rules loading when they shouldn't
  - Diagnosis: Missing EXCLUDE keywords
  - Solution: How to add EXCLUDE patterns

### Integration Issues
- Problem: AGENTS.md integration fails
  - Diagnosis: File permissions or marker conflicts
  - Solution: Manual integration steps
- Problem: opencode.json integration fails
  - Diagnosis: Invalid JSON or permissions
  - Solution: Validate JSON command, fix permissions

Each solution must include:
- Exact command to run
- Expected output
- What to do if it still fails
