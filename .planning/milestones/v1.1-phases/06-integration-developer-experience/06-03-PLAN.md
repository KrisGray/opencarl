<task type="auto">
  <name>Task 1: Create debug logging module</name>
  <files>src/carl/debug.ts</files>
  <action>
Create lightweight debug logging system:

1. Implement `isDebugEnabled(): boolean`:
   - Check `process.env.CARL_DEBUG === 'true'`
   - Cache result at module load for zero-overhead when disabled
   - Returns false if env var not set or not 'true'

2. Implement `debugLog(category: string, message: string, data?: object)`:
   - Early return if !isDebugEnabled() (zero overhead)
   - Format: `[carl:debug] {timestamp} [{category}] {message}`
   - If data provided, append as JSON on next line
   - Use ISO 8601 timestamp format
   - Output via console.log (stdout, not stderr)

3. Implement specialized debug helpers:
   - `debugRuleMatch(domain: string, keywords: string[], matched: string[], excluded: string[])`:
     - Log rule matching decision with matched/excluded keywords
   - `debugInjection(domains: string[], totalRules: number, bracket: string)`:
     - Log injection event with domains being injected and context bracket
   - `debugFileLoad(path: string, scope: string, success: boolean)`:
     - Log file load events (manifest, domain files)

All functions must have zero overhead when debug disabled (check isDebugEnabled() first thing).
  </action>
  <verify>
    <automated>CARL_DEBUG=true node -e "const d = require('./dist/carl/debug.js'); d.debugLog('test', 'hello', {a: 1})"</automated>
  </verify>
  <done>Debug module exports isDebugEnabled, debugLog, and specialized helpers</done>
</task>

<task type="auto">
  <name>Task 2: Add debug logging to loader, matcher, and hooks</name>
  <files>src/carl/loader.ts, src/carl/matcher.ts, src/integration/plugin-hooks.ts</files>
  <action>
Integrate debug logging at key decision points:

1. In loader.ts:
   - Import debugFileLoad
   - Log when manifest files are loaded (success/failure)
   - Log when domain files are loaded (scope, domain name)
   - Log rule count per domain when loaded

2. In matcher.ts (matchDomainsForTurn function):
   - Import debugRuleMatch
   - For each domain evaluated, log:
     - Domain name
     - Keywords found in prompt/signals
     - Whether matched or excluded
     - Which keyword caused match/exclude

3. In plugin-hooks.ts (experimental.chat.system.transform):
   - Import debugInjection
   - After matchDomainsForTurn, log:
     - All matched domains
     - Total rules being injected
     - Current context bracket
   - Log before injection (if any)

Pattern: Keep logging concise but comprehensive enough to answer "why did this rule load?"
  </action>
